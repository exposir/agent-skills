# Skill 与 Rules 的关系说明

根据项目结构设计，**Skill** 和 **Rules** 的关系可以理解为 **“编译产物”与“源代码”** 的关系，或者“整体产品”与“组成部件”的关系。

## 核心概念

### 1. Rules (规则) —— “源代码”

- **位置**：位于 `rules/` 目录下。
- **定义**：每一个 `.md` 文件（如 `async-parallel.md`）代表一条具体的、独立的 React 最佳实践知识点。它是构成 Skill 的原子单元。
- **目的**：这种拆分结构是为了**方便人类维护**。
  - 当你需要修改或新增某条知识时，只需要操作一个小的 Markdown 文件，而不需要在一个巨大的文档中查找。

### 2. Skill (技能) —— “最终产物”

- **位置**：体现为生成的 `AGENTS.md` (或 `SKILL.md`) 文件。
- **定义**：Agent（AI 代理）实际加载和使用的“完整能力说明书”。
- **生成方式**：通过运行 `pnpm build` 命令，脚本会将 `rules/` 目录下的所有独立 Rules 及其元数据，**编译（合并）** 成这一个单一的、结构化的文件。

## 总结

- **Rules** = 组成 Skill 的原材料（拼图碎片）。
- **Skill** = 由 Rules 拼装而成的完整知识体系（完成的拼图）。

## 阅读/维护指南

| 你的身份            | 你应该关注的文件            | 为什么                                                                                                                                         |
| :------------------ | :-------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| **使用者 / 阅读者** | `AGENTS.md` (或 `SKILL.md`) | 因为它包含了所有的 Rules 内容，而且是按照章节整理好的完整文档。`rules/` 目录里的内容在这里都有，无需重复阅读。                                 |
| **维护者 / 开发者** | `rules/` 目录               | 因为这里是“源代码”。如果你要新增规则或修复错误，**必须**修改这里的源文件，然后重新运行构建命令来更新 Skill 文件。直接修改 Skill 文件会被覆盖。 |
