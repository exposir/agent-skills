# Sections

该文件定义了所有章节、顺序、影响级别和描述。
章节 ID（在括号中）是用于对规则进行分组的文件名前缀。

---

## 1. 消除瀑布流 (async)

**影响：** 严重
**描述：** 瀑布流是头号性能杀手。每个顺序的 await 都会增加完整的网络延迟。消除它们可以获得最大的收益。

## 2. 包体积优化 (bundle)

**影响：** 严重
**描述：** 减少初始包体积可以改善交互时间 (TTI) 和最大内容绘制 (LCP)。

## 3. 服务端性能 (server)

**影响：** 高
**描述：** 优化服务端渲染和数据获取可以消除服务端瀑布流并减少响应时间。

## 4. 客户端数据获取 (client)

**影响：** 中-高
**描述：** 自动去重和高效的数据获取模式可以减少多余的网络请求。

## 5. 重渲染优化 (rerender)

**影响：** 中
**描述：** 减少不必要的重渲染可以最大限度地减少浪费的计算并提高 UI 响应能力。

## 6. 渲染性能 (rendering)

**影响：** 中
**描述：** 优化渲染过程可以减少浏览器需要做的工作。

## 7. JavaScript 性能 (js)

**影响：** 低-中
**描述：** 针对热点路径的微优化可以累积成有意义的改进。

## 8. 高级模式 (advanced)

**影响：** 低
**描述：** 针对需要仔细实现的特定情况的高级模式。
